# 1. ОБЩАЯ ИДЕЯ

Приложение — это система для фиксации, анализа и мотивации физической активности пользователя.

Основной фокус:
- трекинг активности
- формирование регулярной привычки
- мотивация через прогресс и бонусы
- социальное сравнение (как вторичный слой)

---

# 2. ОСНОВНОЙ ПРИНЦИП

Центральная единица системы — **ExerciseLog**.

Всё поведение системы строится вокруг факта выполненного упражнения.

---

# 3. МОДЕЛЬ АКТИВНОСТИ ПОЛЬЗОВАТЕЛЯ

## 3.1 Основное действие
- любое добавление ExerciseLog = факт активности пользователя

## 3.2 День пользователя

День может находиться в одном из состояний:

- **Active day** → есть хотя бы один ExerciseLog → влияет на streak
- **Rest day** → день закрыт через бонус отдыха → НЕ ломает streak
- **Missed day** → нет активности и нет бонуса → ломает streak

---

# 4. EXERCISE СИСТЕМА

## 4.1 Структура упражнений

- **Exercise Category**
    - например: отжимания

- **Exercise Type**
    - конкретная вариация внутри категории
    - например: узкий хват, широкий хват, хлопок и т.д.

---

## 4.2 Пользовательские упражнения

- пользователь может создавать новый Exercise Type внутри категории
- сначала он локальный
- затем может быть предложен в общую базу
- попадание в глобальную базу происходит через модерацию (на старте)

---

## 4.3 Статистика упражнений

- статистика считается:
    - по Exercise Type
    - по Category (агрегация типов)

---

# 5. EXERCISE LOG (ЯДРО СИСТЕМЫ)

- каждое выполненное упражнение фиксируется как ExerciseLog
- ExerciseLog существует независимо (не обязан принадлежать тренировке)
- может быть добавлен в контекст тренировки или без неё

---

# 6. ТРЕНИРОВКИ

## 6.1 Природа тренировок

- тренировка — опциональная группировка упражнений
- не обязательна для использования системы

## 6.2 Поведение пользователя

Пользователь может:
- использовать тренировки
- или просто добавлять упражнения напрямую в день

## 6.3 Роль ExerciseLog

- ExerciseLog всегда является самостоятельной сущностью
- тренировка — только логическая группировка

---

# 7. ШАБЛОНЫ ТРЕНИРОВОК

## 7.1 Назначение
- быстрый способ заполнения активности

## 7.2 Природа
- шаблоны = UI-ускоритель
- не являются источником логики или аналитики

## 7.3 Поведение
- пользователь может:
    - использовать готовые шаблоны
    - создавать свои шаблоны
- предложения значений основаны только на статистике прошлых записей

---

# 8. СРЕДНИЕ ЗНАЧЕНИЯ И ПРОГРЕССИЯ

## 8.1 Расчёт среднего

- основа: последние 3 записи
- уровень приоритетов:
    1. Exercise Type
    2. Exercise Category (fallback)

## 8.2 Ограничения

- если < 3 записей → ограничений нет
- среднее используется для:
    - рекомендаций
    - проверки аномалий

---

# 9. ПРОГРЕССИЯ И АНОМАЛИИ

- у каждого Exercise Type есть допустимый max_value без подтверждения
- если значение сильно превышает среднее:
    - система запрашивает подтверждение (например видео)
- цель — проверка достоверности, а не блокировка пользователя

---

# 10. СИСТЕМА СТРИКА

## 10.1 Правило streak

- streak увеличивается при любом ExerciseLog в день

## 10.2 Мягкие пропуски

- доступно ~2 бонусных дня в месяц
- бонус позволяет сохранить streak при отсутствии активности

## 10.3 Дополнительные бонусы

- за регулярность (например 6 дней подряд)
- за объём активности (концепт)

---

# 11. МОТИВАЦИОННАЯ СИСТЕМА

Система избегает давления и строится на гибкости:

- регулярные напоминания
- акцент на выгоде регулярности
- бонусы вместо наказаний

---

## 11.1 Бонусы

Могут включать:
- пропуск дня без потери streak
- автоматическое добавление тренировки
- “закрытие прошлого дня” (редкий бонус)

---

## 11.2 Автоматическое добавление тренировки

- считается полноценной активностью
- влияет на streak
- может использовать шаблон как основу

---

# 12. СРЕДА РАЗВИТИЯ УПРАЖНЕНИЙ

- система поддерживает расширение упражнений пользователями
- развитие идёт через:
    - локальное создание
    - предложение в глобальную базу
    - модерацию

---

# 13. СОЦИАЛЬНАЯ СИСТЕМА

## 13.1 Роль

- вторичный слой над core системой
- не влияет напрямую на ExerciseLog и core данные

## 13.2 Сравнение пользователей

- на старте: визуальная аналитика
- в будущем: может влиять на:
    - рекомендации
    - статистические инсайты

## 13.3 Приватность

- управляет видимостью данных
- не блокирует участие в агрегированной статистике

---

## 13.4 Социальный поиск

Пользователи могут:
- находить других по активности
- фильтровать по упражнениям или параметрам
- добавлять в друзья

---

# 14. ОБЩАЯ ПРОДУКТОВАЯ ЛОГИКА

Система строится вокруг цикла:

1. пользователь фиксирует активность (ExerciseLog)
2. получает обратную связь (статистика / streak)
3. получает мотивацию (бонусы / сравнение)
4. продолжает активность
